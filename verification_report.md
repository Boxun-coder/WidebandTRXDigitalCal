# AD9164-FMCC-EBZ Bring-Up Verification Template

## Scope
This document is a pre-populated verification worksheet for the custom ADS7-V2EBZ integration. It is intentionally written before bench validation so the measurement flow is ready once the physical connections are complete.

## Preconditions
- Vivado bitstream generated from `build_project.tcl`
- Exported hardware handoff `.xsa`
- Vitis application built from `main.c`
- FMC cable/board stack assembled with AD9164-FMCC-EBZ attached
- Oscilloscope, probe, and logic-voltage measurement tools available

## Stage 1: Safe Digital Bring-Up
- [ ] Power the ADS7-V2EBZ and halt the software before the first AD9164 SPI write.
- [ ] Confirm the board I/O bank used for `uart_ext_txd`, `uart_ext_rxd`, and `scope_trig` is configured for 1.8 V in hardware.
- [ ] Measure `uart_ext_txd` idle voltage and record the observed level.
- [ ] Measure `uart_ext_rxd` idle voltage and record the observed level.
- [ ] Confirm `scope_trig` pulses when the firmware starts a chirp burst.
- [ ] Open the debug checkpoint generated by `system_ila.tcl`.
- [ ] Verify `ila_sync` toggles as expected during JESD link establishment.
- [ ] Verify `ila_sysref` is present and clean.
- [ ] Verify `ila_chirp_tvalid` asserts and `ila_chirp_tdata` changes during a chirp burst.
- [ ] Capture screenshots of the ILA traces for the project log.

## Stage 2: JESD204B Link Validation
- [ ] Read back AD9164 registers `0x470` through `0x473` and store the values.
- [ ] Confirm CGS status indicates all lanes aligned.
- [ ] Confirm frame alignment and checksum bits are all set.
- [ ] Note any lane deskew or ILAS anomalies.

## Stage 3: RF Verification
- [ ] Connect the AD9164 RF output to a high-bandwidth oscilloscope input.
- [ ] Connect `scope_trig` to the oscilloscope external trigger input.
- [ ] Trigger on the chirp marker pulse.
- [ ] Capture the time-domain RF envelope.
- [ ] Run an STFT or spectrogram measurement.
- [ ] Compare the measured sweep slope against the MATLAB reference in `chirp_gen.m`.
